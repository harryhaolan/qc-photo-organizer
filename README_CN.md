# 产品质检拍照整理

For English, see [README.md](./README.md).

一个零构建、纯前端的网页工具，用于产品质检拍照存档。质检人员填写产品型号与编号，为 11 个固定部位分别拍照（可选），对存在瑕疵的部位勾选标记并填写说明，然后一键导出命名规范、可直接归档的文件夹（以 ZIP 形式下载）。全部逻辑在浏览器内完成，无后端、无构建步骤。

界面为简体中文（面向质检与工厂人员）。

## 功能

- 11 个固定部位槽位，分为外部（5）与内部（6）；每个槽位的照片均为可选。
- 每个部位带「有瑕疵」勾选框：仅在勾选后才能填写瑕疵备注，且被勾选的照片会额外复制一份到瑕疵文件夹，并按外部/内部归类。
- 统一的文件与文件夹命名，便于溯源（见下方命名规则）。
- 生成 `质检备注.csv` 清单（UTF-8 带 BOM），记录每个部位的：是否有照片、是否有瑕疵、文件路径、瑕疵备注；可用 Excel、WPS、Numbers 直接打开。
- 一键导出：打包成单一顶层文件夹的 ZIP 并下载（使用 [JSZip](https://stuk.github.io/jszip/)）。
- 「下一台」：保留型号，编号自动加一（01 到 02），并清空表单以录入下一台。
- 本次记录面板：查看本会话已生成的每一台（缩略图与备注），并可重新下载。
- 手机与电脑都适配：响应式栅格、超大触摸区域、手机可直接调用相机、桌面支持拖拽、动画流畅，并支持系统的减弱动态效果。

## 11 个部位

| 外部（5） | 内部（6） |
| --- | --- |
| 正面 / 背面 / 左侧板 / 右侧板 / 顶板 | 内侧板（左）/ 内侧板（右）/ 内背板 / 坐板 / 坐前板 / 脚板 |

## 命名规则

以型号 `ABC`、编号 `01` 为例，导出 `ABC-01.zip`，解压后为：

```
ABC-01/                          单台文件夹 = 型号-编号
├── 外部/                         外部各面（已拍摄的全部照片）
│   ├── ABC-01-正面.jpg           照片 = 型号-编号-部位.扩展名
│   └── ...
├── 内部/                         内部各面（已拍摄的全部照片）
│   └── ...
├── 瑕疵/                         仅包含被勾选的照片（副本）
│   ├── 外部/
│   │   └── ABC-01-正面.jpg       因为「正面」被勾选有瑕疵
│   └── 内部/
│       └── ABC-01-坐板.jpg
└── 质检备注.csv                  备注清单（始终生成）
```

被勾选的照片会出现两次：一次在原本的 `外部`/`内部` 位置，一次在 `瑕疵/外部` 或 `瑕疵/内部`。只有包含照片的子文件夹才会被创建。

`质检备注.csv` 列为：`类别, 部位, 是否有照片, 是否有瑕疵, 文件名, 瑕疵备注`，前面附有型号、编号、生成时间、部位照片数量、瑕疵项数的表头。

型号与编号会做文件名安全处理（`/ \ : * ? " < > |` 等字符替换为 `_`）；中文字符以 UTF-8 完整保留在 ZIP 条目名中。

## 本地运行

这是一个静态网站，无需安装、无需构建。

- 最简单：直接用浏览器打开 `index.html`。
- 推荐（避免个别浏览器对 `file://` 的限制）：用本地服务器打开，例如：

  ```bash
  cd qc-photo-organizer
  python3 -m http.server 8000
  # 然后打开 http://localhost:8000
  ```

## 部署到 Vercel

本目录自带零构建的 `vercel.json`（无构建命令；直接托管静态文件；附带缓存与安全响应头）。

方式 A —— Vercel CLI，在本目录执行：

```bash
cd qc-photo-organizer
npx vercel          # 预览部署
npx vercel --prod   # 生产部署
```

方式 B —— Git 导入：在 Vercel 后台「Add New Project」，Framework Preset 选「Other」，构建命令留空。若从 monorepo 部署，将 Root Directory 设为 `qc-photo-organizer`。

## 已知限制

- iOS Safari 下载：较旧的 iOS 可能忽略 ZIP 文件名，改为在新标签页或分享面板中打开，而非直接保存。如未自动下载，请使用浏览器的下载或分享功能保存。
- HEIC 照片：iPhone「高效」格式拍出的是 HEIC，多数桌面浏览器无法显示，这类槽位会显示占位提示，但原始文件仍会以 `.heic` 扩展名打进 ZIP。本工具不内置 HEIC 解码库。若需要预览，可把手机相机格式设为「兼容性最佳」（JPEG）。
- 记录仅限本次会话：记录面板存在内存中，刷新页面即清空。
- 旧解压工具的中文文件名：条目名是合法 UTF-8，在 macOS 访达/归档实用工具、Windows 10+ 资源管理器、7-Zip、WPS 中都正常显示。仅老旧的 Info-ZIP `unzip` 命令行（及 Windows 7 资源管理器）会忽略 UTF-8 标志、可能显示乱码，请改用现代工具解压。
