# 产品质检拍照整理

> For English, see [README.md](./README.md)

一个**零构建**的纯前端网页工具，用于**产品质检拍照存档**。质检人员填写产品型号 + 编号，为 **11 个固定部位**分别（可选）上传/拍照并填写**瑕疵备注**，还可添加任意多张**瑕疵特写照片**，一键导出一个命名规范、可直接归档的文件夹（以 ZIP 形式下载）。全部逻辑在**浏览器内**完成 —— 无后端、无构建步骤。

界面为简体中文（面向质检 / 工厂人员）。

## 功能

- 📸 **11 个固定部位槽位**，分为 *外部 5 面* 与 *内部 6 面*；每个槽位的照片**都可选**。
- 🔎 **瑕疵特写照片** —— 可添加任意多张缺陷特写，每张可写说明，导出到 `瑕疵` 子文件夹。
- 🏷️ **统一命名**，方便溯源（见下方命名规则）。
- 📝 **每个部位单独的瑕疵备注**，导出为 `质检备注.csv`（UTF-8 带 BOM，Excel / WPS / Numbers 直接打开不乱码）。
- 📦 **一键导出** —— 打包成单一顶层文件夹，并按 `外部` / `内部` / `瑕疵` 分成子文件夹，下载为 ZIP（使用 [JSZip](https://stuk.github.io/jszip/)）。
- ➡️ **「下一台」** —— 保留型号，编号自动 +1（`01 → 02`），清空照片，马上拍下一台。
- 🗂️ **本次记录** —— 查看本次会话已生成的每一台（缩略图 + 备注），并可重新下载。
- 📱 **手机 / 电脑都好用** —— 响应式栅格、超大触摸区域、手机可直接调用相机、桌面支持拖拽、动画流畅，并支持系统的「减弱动态效果」。

## 11 个部位

| 外部 5 面 | 内部 6 面 |
| --- | --- |
| 正面 / 背面 / 左侧板 / 右侧板 / 顶板 | 内侧板（左）/ 内侧板（右）/ 内背板 / 坐板 / 坐前板 / 脚板 |

## 命名规则

以型号 `ABC`、编号 `01` 为例，导出 `ABC-01.zip`，解压后为：

```
ABC-01/                        ← 文件夹 = 型号-编号
├── 外部/                       ← 外部 5 面
│   ├── ABC-01-正面.jpg         ← 照片 = 型号-编号-部位.扩展名
│   └── …
├── 内部/                       ← 内部 6 面
│   └── …
├── 瑕疵/                       ← 瑕疵特写（数量不限）
│   ├── ABC-01-瑕疵-01.jpg      ← 型号-编号-瑕疵-序号.扩展名
│   └── …
└── 质检备注.csv                ← 备注清单（始终生成）
```

只有「有照片」的子文件夹才会被创建。

`质检备注.csv` 列为：`类别, 部位/序号, 是否有照片, 文件名, 备注`，前面附有 型号 / 编号 / 生成时间 / 部位照片 / 瑕疵照片 的表头。`文件名` 列是相对单台文件夹的路径（如 `外部/ABC-01-正面.jpg`）。

型号 / 编号会做文件名安全处理（非法字符 `/ \ : * ? " < > |` 等替换为 `_`）；**中文字符会完整保留**（ZIP 条目名为 UTF-8）。

## 本地运行

这是一个静态网站 —— 无需安装、无需构建。

- **最简单：** 直接用浏览器打开 `index.html`。
- **推荐（避免个别浏览器对 `file://` 的限制）：** 用本地服务器打开，例如：

  ```bash
  cd qc-photo-organizer
  python3 -m http.server 8000
  # 然后打开 http://localhost:8000
  ```

## 部署到 Vercel

本目录自带零构建的 `vercel.json`（无构建命令；直接托管静态文件；附带缓存 / 安全响应头）。

**方式 A —— Vercel CLI**（在本目录执行）：

```bash
cd qc-photo-organizer
npx vercel          # 预览部署
npx vercel --prod   # 生产部署
```

**方式 B —— Git 导入：** 把仓库推到 GitHub，在 Vercel 后台「Add New Project」，将 **Root Directory 设为 `qc-photo-organizer`**（若是独立仓库则用仓库根目录），Framework Preset 选 **Other**，构建命令留空。

## 已知注意事项

- **iOS Safari 下载：** 较旧的 iOS 可能忽略 ZIP 文件名，改为在新标签页 / 分享面板中打开，而非直接保存。如未自动下载，请使用浏览器的下载 / 分享功能保存。
- **HEIC 照片：** iPhone「高效」格式拍出的是 HEIC，多数桌面浏览器无法显示 —— 这类槽位会显示「无法预览」占位，**但原始文件仍会以 `.heic` 扩展名打进 ZIP**。本工具不内置 HEIC 解码库（刻意保持简单）。若需要预览，可把手机相机格式设为「兼容性最佳 / JPEG」。
- **记录仅限本次会话：** 记录面板存在内存中，刷新页面即清空。（如需跨刷新保存，后续可加 IndexedDB。）
- **旧解压工具的中文文件名：** 条目名是合法 UTF-8，在 macOS 访达 / 归档实用工具、Windows 10+ 资源管理器、7-Zip、WPS 中都正常显示。仅老旧的 Info-ZIP `unzip` 命令行（及 Windows 7 资源管理器）会忽略 UTF-8 标志、可能显示乱码 —— 请改用现代工具解压。
